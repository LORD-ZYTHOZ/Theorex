  <behavior>
    isImportant cases (the 6 gate rules):
    - Rule 1: canonicalForm.length <= 2 AND !isMultiWord → false (e.g., "it", "go")
    - Rule 2: STOP_CONCEPTS.has(canonicalForm) → false (e.g., "thing", "system", "people")
    - Rule 3: tags includes "Person", "Place", "Organization", or "Value" → true (named entities always pass)
    - Rule 4: isMultiWord === true → true (e.g., "machine learning", "New York")
    - Rule 5: tags includes "Acronym" → true (e.g., "NASA", "ML")
    - Rule 6: tags includes "ProperNoun" → true (e.g., "TypeScript", "Python")
    - Default: single-word common noun, no special tag → false

    importanceGate cases:
    - input: IdentifiedConcept[] with mixed important and unimportant concepts → output contains ONLY important ones
    - input: empty array → output: empty array
    - output objects have gatePass: true and importanceScore: 1.0
    - output is a new array — input array is not mutated

    amplifyFrequency cases:
    - input: GatedConcept with canonicalForm "typescript", originalText "TypeScript is great. TypeScript scales." → frequencyCount=2, frequencyAmplifier=1+ln(2)≈1.693
    - input: GatedConcept with canonicalForm "typescript", originalText with 1000 occurrences → frequencyAmplifier = 1+ln(1000)≈7.908 (log-normalized, not linear)
    - input: empty GatedConcept array → output: empty array
    - input: concept with canonicalForm containing regex special chars (e.g. "c++") → no crash (regex escaped)
    - output objects spread all GatedConcept fields + add frequencyCount and frequencyAmplifier (immutable)
    - amplifyFrequency() signature: (gated: readonly GatedConcept[], originalText: string) — TypeScript REJECTS passing IdentifiedConcept[] here

    STOP_CONCEPTS must include at minimum:
    "thing", "way", "time", "people", "year", "day", "man", "woman", "child", "world", "life",
    "hand", "part", "place", "case", "week", "company", "system", "program", "question", "work",
