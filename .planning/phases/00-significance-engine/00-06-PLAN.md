  <behavior>
    processText cases:
    - input: same text + same sourceWeight twice → JSON.stringify(a) === JSON.stringify(b) (byte-identical)
    - input: text with "system" repeated 1000 times → output is [] (gate rejects "system"; amplify never runs for it)
    - input: "TypeScript " + "thing ".repeat(100) → output includes TypeScript event with composite_score > 0; "thing" is absent
    - input: sourceWeight=0.5, named entity "TypeScript" → all output events have source_weight=0.5
    - input: sourceWeight=1.0 → composite_score = importanceScore × frequencyAmplifier × 1.0
    - input: empty string "" → output: []
    - input: text with only stop words → output: []
    - output array is sorted by concept_id ascending — deterministic order

    composite_score formula verified by test:
    - For a concept with importanceScore=1.0, frequencyCount=3, sourceWeight=0.7:
      expected composite_score = 1.0 × (1 + Math.log1p(3)) × 0.7 ≈ 0.973

    Purity confirmed by test:
    - No global state is modified between calls (call processText 3 times, all produce same result)

    attachMetadata behavior:
    - Builds ConceptEvent from ScoredConcept + input.sourceWeight + input.nodeType + input.timestamp
    - concept_id = scored.conceptId
    - surface_form = scored.surfaceForm
    - importance_score = scored.importanceScore
    - frequency_count = scored.frequencyCount
    - composite_score = scored.importanceScore × scored.frequencyAmplifier × input.sourceWeight
    - source_weight = input.sourceWeight (caller-injected, not computed)
    - node_type = input.nodeType
    - timestamp = input.timestamp (caller-injected — NOT Date.now() inside compose)

    processText signature (convenience wrapper over PipelineInput):
    processText(text: string, sourceWeight: number, nodeType: NodeType = "concept", timestamp?: string): readonly ConceptEvent[]
    — if timestamp omitted, use new Date().toISOString() as default at call time (outside pipeline pure core)
  </behavior>
