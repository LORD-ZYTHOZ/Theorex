    - else → "LESS"

    propagate.ts — mutates AxonStore graph:

    propagateActivation(store: AxonStore, nodeKey: string, activationDelta: number, nowMs: number): void
    - Increments activated node frequency_count by 1, updates last_seen to new Date(nowMs).toISOString()
    - Collects neighbors BEFORE mutating: const neighbors = store.graph.neighbors(nodeKey)
    - For each neighbor: importance_weight += activationDelta × 0.5, clamped to 1.0
    - NO further iteration — strictly one hop
    - Second-hop nodes are completely untouched (verify in tests)

    propagateSentiment(store: AxonStore, nodeKey: string, sentiment: "PREFERRED" | "DISPREFERRED", nowMs: number): void
    - Sets sentiment_tier on nodeKey
    - Collects neighbors before mutating
    - For each neighbor: if sentiment is PREFERRED, nudge importance_weight up by 0.05; if DISPREFERRED, nudge down by 0.05 (clamped 0.0–1.0). Do NOT change neighbor's sentiment_tier itself — only importance_weight is affected
    - One hop only

    Note on SNT-04: A node can be ACTIVE + DISPREFERRED. relevance_tier and sentiment_tier are fully independent fields. Propagate functions never couple them.

    ```typescript
    export function propagateActivation(
      store: AxonStore,
      nodeKey: string,
      activationDelta: number,
      nowMs: number,
    ): void {
      const g = store.graph;
      g.updateNodeAttribute(nodeKey, "frequency_count", (n) => (n ?? 0) + 1);
      g.updateNodeAttribute(nodeKey, "last_seen", () => new Date(nowMs).toISOString());

      // Collect BEFORE mutating — mutation during iteration is unsafe
      const neighbors = g.neighbors(nodeKey);
      const dampened = activationDelta * 0.5;
      for (const neighbor of neighbors) {
        g.updateNodeAttribute(neighbor, "importance_weight", (w) =>
          Math.min(1.0, (w ?? 0) + dampened)
        );
      }
      // No second-hop — stop here
    }
    ```
